Короче, я могу использовать идею семестралки дбс, 
    внедрив также энтити работников, которым будут приделяться запросы


================= **ENTITIES** ================= 
* работник:
    - имя
    - фамилия
    - id
    - id заявки на рассмотрении и т.д.

* юзер:
    - имя
    - фамилия
    - номер_телефона
    - перечень запросов на возврат вещей
    - почта

* вещь:
    - id
    - название
    - категории
    - краткое описание
    - фото
    - место находки (тип транспорта, номер маршрута и номер транспортного средства)
    - текущее место нахождения (будет видно пользователю только после подтверждения)

* склад:
    - id склада
    - адрес


================= **DATA** ================= 

* запрос возврата:
    - id запроса
    - статус подтверждения
    - дата подтверждения

* категория:
    - id категории
    - название категории

* страны доставки


================= **SERVICES** ================= 

* сервис поиска:
    - по категории
    - по названию

* сервис подтверждения

* сервисы выдачи потерянных предметов
    - особни вызведнути на складе
    - доставка
        - доставка внутри страны
        - доставка заграницу 
                (можно дёргать типо API какого-то UPS службы доставки или типо того)
